## 系统概述

Recipe Agent 采用典型的分层架构设计，从用户接入到底层数据存储，各层职责清晰，易于扩展和维护。

## 核心组件

### 1. 用户接入层

- **飞书 Bot：** 通过飞书开放平台接入
- **微信 Bot：** 通过微信公众号/企业微信接入
- **钉钉 Bot：** 通过钉钉开放平台接入
- **Web/App：** 自建的 Web 和移动端应用

### 2. API 网关层

- **FastAPI：** 高性能 Python Web 框架
- **认证：** JWT/OAuth 2.0
- **限流：** 防止 API 滥用
- **审计：** 记录所有 API 请求

### 3. Agent 编排层

- **LangChain：** Agent 编排框架
- **Task Routing：** 任务路由分发
- **Skills：** 模块化技能组件
- **State Management：** 会话状态管理

### 4. 多模态预处理

- **图像识别：** ViT/ResNet/CLIP
- **视频分析：** 关键帧提取 + 动作识别
- **ASR：** Whisper 语音转文字

### 5. LLM 适配层

- **deepseek v3.2：** 主要 LLM
- **seed 2.0：** 备选 LLM
- **Adapter：** 统一的 LLM 调用接口
- **Fallback：** 故障转移机制

### 6. 规则与校验

- **营养核验：** 营养成分验证
- **特殊人群规则：** 儿童/老人/孕妇/病患
- **中医食疗规则：** 节气/体质配方

### 7. 数据存储层

- **Postgres：** 关系型数据存储
- **Redis：** 缓存和会话存储
- **VectorDB：** 向量数据库（Chroma/FAISS）
- **S3：** 对象存储（图片/视频）

## 数据流

### 文本菜谱生成流程

```
用户输入文本
    │
    ▼
API 网关（认证/限流/审计）
    │
    ▼
Agent 编排层
    │
    ├─→ 个性化画像查询（Redis/Postgres）
    │
    ├─→ LLM 适配层（deepseek/seed）
    │
    ├─→ 营养计算（规则引擎）
    │
    └─→ 结果返回
```

### 图片菜谱生成流程

```
用户上传图片
    │
    ▼
API 网关（认证/限流/审计）
    │
    ▼
Agent 编排层 → 创建异步任务
    │
    ├─→ 多模态预处理（图像识别）
    │       │
    │       ├─→ 图片存储（S3）
    │       └─→ 食材识别（ViT/CLIP）
    │
    ├─→ LLM 适配层（菜谱生成）
    │
    ├─→ 营养计算（规则引擎）
    │
    └─→ 任务完成通知（Webhook/推送）
```

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| **后端框架** | Python 3.10+, FastAPI |
| **Agent 编排** | LangChain (Python) |
| **云模型** | deepseek v3.2 / seed 2.0 |
| **视觉/视频** | PyTorch + Hugging Face 模型 |
| **ASR** | Whisper |
| **知识库** | LlamaIndex + Chroma/FAISS |
| **DB/存储** | Postgres + Redis + S3 |
| **部署** | Docker + GitHub Actions, K8s |
| **监控** | Prometheus + Grafana |
| **日志** | ELK / Loki |
