## 一、产品概述（Why）

### 1.1 产品概要

基于多模态 LLM（deepseek v3.2 / seed 2.0）与本地视觉/音频能力，打造一款"从图文/视频/语音直接生成可执行菜谱、营养与中医食疗分析、特殊人群调整、视频脚本输出及外部餐饮/电商推荐"的智能 Agent。

### 1.2 核心价值

- **节省时间：** 快速确定"今天吃什么"的决策时间
- **提升可复制性：** 家庭/社群烹饪标准化
- **健康场景：** 个性化饮食建议
- **变现渠道：** 电商、餐厅推荐、内容变现

### 1.3 产品定位

智能饮食健康管理与内容创作一体化平台

## 三、目标用户（Personas）

### 3.1 忙碌上班族

**画像：** 25-40岁，工作繁忙，希望快速获得健康、快速的饮食方案

**核心需求：**
- 快速候选菜谱（3-5个）
- 一键生成购物清单
- 30分钟内可完成的菜谱
- 营养均衡的早餐/午餐/晚餐

### 3.2 健康管理者/患者

**画像：** 关注健康或有特定健康需求的人群

**核心需求：**
- 基于疾病/过敏的安全菜谱
- 低钠/低糖/低脂等特殊需求
- 营养成分详细标注
- 医生/营养师推荐

### 3.3 中医养生用户

**画像：** 关注中医养生，希望通过饮食调理身体

**核心需求：**
- 节气/体质配方
- 中医理论支持
- 食材性味归经
- 养生功效说明

### 3.4 社群管理员

**画像：** 微信群/社区管理者，希望活跃群聊氛围

**核心需求：**
- 群聊投票功能
- 群内食材采集
- 群分享功能
- 菜谱分享与讨论

### 3.5 内容创作者/厨师

**画像：** 美食博主、厨师，希望快速生成内容

**核心需求：**
- 一键生成视频脚本
- 自动导出字幕
- 专业排版
- 多平台适配

## 四、关键功能（MVP 优先级）

### MVP（第一轮必须实现）

#### 1. 文本输入 → 结构化菜谱输出（JSON schema）

- 支持自然语言输入
- 生成结构化 JSON 输出
- 包含食材、步骤、时间、难度等
- 支持多语言

#### 2. 个性化口味画像与偏好记忆（加密存储）

- 用户口味偏好记录
- 饮食禁忌记录
- 历史记录查询
- 数据加密存储

#### 3. 营养基础计算（卡路里/三大营养素）

- 卡路里计算
- 蛋白质/脂肪/碳水化合物
- 维生素/矿物质
- 每日推荐摄入量

#### 4. "今天吃什么"快速候选（3-5 个）

- 智能推荐
- 基于历史偏好
- 基于季节
- 基于食材库存

#### 5. 飞书/微信/钉钉文本 Bot（收/发文本与图片）

- 多平台支持
- 文本交互
- 图片识别
- 消息推送

#### 6. 权限与隐私策略（默认本地加密）

- 数据加密
- 权限控制
- 隐私设置
- 数据导出

### MVP+（第二轮优先）

#### 1. 图片 → 菜谱（食材识别 + 置信度）

- 图片上传
- 食材识别
- 置信度显示
- 用户确认

#### 2. 视频 → 菜谱（关键帧 → 步骤映射 + video_script）

- 视频上传
- 关键帧提取
- 步骤映射
- 视频脚本生成

#### 3. 特殊人群规则（儿童/老人/孕妇/病患）

- 儿童食谱
- 老人食谱
- 孕妇食谱
- 病患食谱

#### 4. 中医节气食疗规则集 + 文献引用

- 节气食谱
- 体质调理
- 文献引用
- 专家推荐

### 长期（第三轮及以后）

#### 1. 餐厅（大众点评/Google/Yelp）对接，外部就餐推荐

- 餐厅推荐
- 菜品推荐
- 优惠信息
- 预订功能

#### 2. 电商对接（京东/淘宝/Amazon）支持一键购菜

- 一键购菜
- 价格对比
- 配送信息
- 优惠券

#### 3. 本地化离线模型与混合推理（云 + 本地）

- 离线模型
- 混合推理
- 隐私保护
- 性能优化

#### 4. 高级菜单计划与库存管理

- 周计划
- 月计划
- 库存管理
- 自动补货

### 6.1 输入规范

| 类型 | 格式 | 说明 |
|------|------|------|

- recipe JSON（见 schema）
- video_script JSON
- analysis_report

### 6.3 交互模式

- 同步快速响应（文本候选）
- 异步多模态任务（图片/视频—任务完成后通知用户）

## 十、风险与缓解

### 10.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|------------|
| 视频逆向生成不稳定 | 高 | 中 | 先限定输入规格与最低置信度；将低置信度标注并要求用户确认 |
| 模型成本过高 | 中 | 高 | 模型适配层，支持多种模型切换；成本优化策略 |
| 系统性能瓶颈 | 中 | 中 | 缓存策略；异步处理；弹性伸缩 |

### 10.2 业务风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|------------|
| 中医食疗与现代医学冲突 | 高 | 中 | 标注证据等级与免责声明，敏感场景提示就医 |
| 用户接受度低 | 高 | 中 | 用户调研；快速迭代；A/B 测试 |
| 外部平台对接复杂且有费用 | 中 | 高 | 抽象适配层，先实现 mock，再扩展到真实 API |

### 10.3 合规风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|------------|
| 隐私合规问题 | 高 | 中 | 隐私设计；合规审计；用户告知 |
| 医疗建议责任 | 高 | 低 | 明确免责声明；不替代专业医疗；专家审核 |
